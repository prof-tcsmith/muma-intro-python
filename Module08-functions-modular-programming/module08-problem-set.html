Solution 3: Calculating Area (Return Value)

# Define the function with parameters and return statement
def calculate_rectangle_area(length, width):
    """Calculates and returns the area of a rectangle."""
    area = length * width
    return area # Sends the calculated value back

# Call the function and store the returned value
rect_area = calculate_rectangle_area(15.5, 10)

# Expected Output:

Explanation:

  • The function takes length and width as parameters.

  • It calculates the area locally.

  • The return area statement does two things: it exits the function and sends the value stored in the area variable back to where the function was called.

  • The line rect_area = calculate_rectangle_area(15.5, 10) calls the function, and the returned value (155.0) is assigned to the rect_area variable.

Solution 4: print vs. return

# a) Function using print
def print_division(numerator, denominator):
    """Calculates and prints the division result."""
    if denominator != 0: # Basic check to avoid error
        result = numerator / denominator
        print(f"Result of division: {result}")
    else:
        print("Cannot divide by zero.")
    # No explicit return, so it returns None implicitly

# b) Function using return
def return_division(numerator, denominator):
    """Calculates and returns the division result."""
    if denominator != 0:
        result = numerator / denominator
        return result # Return the calculated value
    else:
        print("Cannot divide by zero.")
        return None # Good practice to return None if calculation fails

# Call both functions and capture results
print("Calling print_division(100, 4):")
result_from_print = print_division(100, 4) # This line will print output

print("\nCalling return_division(100, 4):")
result_from_return = return_division(100, 4) # This line will NOT print output

# Print the captured variables
print(f"\nValue stored from print_division: {result_from_print}")
print(f"Value stored from return_division: {result_from_return}")

# Expected Output:
# Calling print_division(100, 4):
# Result of division: 25.0
#
# Calling return_division(100, 4):
#
# Value stored from print_division: None
# Value stored from return_division: 25.0

Explanation:

  • print_division displays the result (25.0) directly to the console when called. However, since it lacks an explicit return statement, it implicitly returns None. Therefore, result_from_print becomes None.

  • return_division calculates the result but does not print it automatically. Instead, it uses return result to send the calculated value (25.0) back. This returned value is then captured and stored in the result_from_return variable.

  • This clearly shows that print is for display, while return is for sending back a value to be used by the calling code.

Solution 5: Tax Calculation with Default Rate

# Define function with a default value for tax_rate
def calculate_sales_tax(amount, tax_rate=0.07):
    """Calculates sales tax, using a default rate if none provided."""
    tax_amount = amount * tax_rate
    return tax_amount

# a) Call using the default tax_rate (0.07)
default_tax = calculate_sales_tax(500.00)
print(f"Tax on $500.00 (default rate): ${default_tax:.2f}")

# b) Call providing a specific tax_rate (0.09)
specific_tax = calculate_sales_tax(500.00, 0.09)
print(f"Tax on $500.00 (specific rate 9%): ${specific_tax:.2f}")

# Expected Output:
# Tax on $500.00 (default rate): $35.00
# Tax on $500.00 (specific rate 9%): $45.00

Explanation:

  • In the definition def calculate_sales_tax(amount, tax_rate=0.07):, tax_rate is given a default value of 0.07. Parameters with defaults must come after those without.

  • When called with only one argument (calculate_sales_tax(500.00)), the amount parameter gets 500.00, and tax_rate automatically uses its default value (0.07).

  • When called with two arguments (calculate_sales_tax(500.00, 0.09)), the provided value 0.09 overrides the default for tax_rate.

Solution 6: Using Keyword Arguments

# Function definition from Problem 5
def calculate_sales_tax(amount, tax_rate=0.07):
    """Calculates sales tax, using a default rate if none provided."""
    tax_amount = amount * tax_rate
    return tax_amount

# a) Call using keyword arguments
tax1 = calculate_sales_tax(amount=1200.00, tax_rate=0.06)
print(f"Tax using keyword args (amount=1200, rate=0.06): ${tax1:.2f}")

# b) Call using keyword arguments in a different order
tax2 = calculate_sales_tax(tax_rate=0.08, amount=500.00)
print(f"Tax using keyword args (rate=0.08, amount=500): ${tax2:.2f}")

# Expected Output:
# Tax using keyword args (amount=1200, rate=0.06): $72.00
# Tax using keyword args (rate=0.08, amount=500): $40.00

Explanation:

  • Keyword arguments use the format parameter_name=value in the function call.

  • This explicitly assigns the value to the named parameter, regardless of its position.

  • As shown in call (b), the order of keyword arguments does not matter (tax_rate is provided before amount). This improves readability, especially for functions with many parameters or optional parameters.

Solution 7: Analyze Sales Data (Return Multiple Values)

import math # Using math just for demonstration, sum/len are built-in

def analyze_sales(sales_figures_list):
    """Calculates total sales, count, and average from a list."""
    if not sales_figures_list: # Check if the list is empty
        return 0.0, 0, 0.0 # Return zeros/default values

    total = sum(sales_figures_list) # Module 5 aggregation
    count = len(sales_figures_list) # Module 5 aggregation
    average = total / count

    return total, count, average # Return multiple values

# Sample data
sales_data = [150.00, 210.50, 180.00, 305.75]

# Call the function and capture the returned tuple
analysis_results = analyze_sales(sales_data)
print(f"Returned tuple: {analysis_results}")

# Unpack the tuple into separate variables
total_sales, num_transactions, avg_sale = analysis_results

# Print the unpacked variables
print(f"Total Sales: ${total_sales:.2f}")
print(f"Number of Transactions: {num_transactions}")
print(f"Average Sale Value: ${avg_sale:.2f}")

# Call with empty list
total_e, count_e, avg_e = analyze_sales([])
print(f"\nAnalysis for empty list: Total=${total_e}, Count={count_e}, Avg=${avg_e}")


# Expected Output:
# Returned tuple: (846.25, 4, 211.5625)
# Total Sales: $846.25
# Number of Transactions: 4
# Average Sale Value: $211.56
#
# Analysis for empty list: Total=$0.0, Count=0, Avg=$0.0

Explanation:

  • The function analyze_sales calculates three values: total, count, and average.

  • The line return total, count, average returns these three values. Python automatically packages them into a tuple: (846.25, 4, 211.5625).

  • The first call captures this entire tuple into the analysis_results variable.

  • The line total_sales, num_transactions, avg_sale = analysis_results uses tuple unpacking (from Module 4) to assign each element of the returned tuple to a corresponding variable on the left side. This is a very common and readable way to handle functions returning multiple results.

  • An if not sales_figures_list: check handles the edge case of an empty list to prevent division by zero.

Solution 8: Variable Scope Illustration

# a) Global variable
global_rate = 0.10
print(f"Initial global rate: {global_rate}")

# b) Function reading global variable
def calculate_discount(price):
    """Calculates discount using the global rate."""
    # Reads global_rate (no 'global' keyword needed for reading)
    discount = price * global_rate
    print(f" Inside calculate_discount: Global rate used = {global_rate}")
    return discount

# c) Function using a local variable with the same name
def apply_local_discount(price):
    """Uses a local variable that shadows the global one."""
    discount_percentage = 0.15 # This is LOCAL, different from the global one
    print(f" Inside apply_local_discount: Local rate used = {discount_percentage}")
    discounted_price = price - (price * discount_percentage)
    return discounted_price

# d) Call function reading global
global_discount = calculate_discount(100)
print(f"Discount using global rate: ${global_discount:.2f}")

# e) Call function using local
local_discounted_price = apply_local_discount(100)
print(f"Price after local discount: ${local_discounted_price:.2f}")

# f) Check global variable value after calls
print(f"Final global rate (unchanged): {global_rate}")

# Expected Output:
# Initial global rate: 0.1
#  Inside calculate_discount: Global rate used = 0.1
# Discount using global rate: $10.00
#  Inside apply_local_discount: Local rate used = 0.15
# Price after local discount: $85.00
# Final global rate (unchanged): 0.1

Explanation:

  • global_rate = 0.10 defines a variable outside any function, making it global.

  • calculate_discount can read global_rate without needing the global keyword.

  • apply_local_discount defines its own variable discount_percentage = 0.15. This is a local variable, distinct from the global one, even though it has the same name (it “shadows” the global within the function). Changes to this local variable do not affect the global one.

  • The final print shows that the global global_rate remained 0.10, demonstrating that the local variable in apply_local_discount did not modify the global scope.

Solution 9: Simple Lambda Function

# Define the lambda function and assign it to a variable
get_double = lambda x: x * 2

# Call the lambda function via the variable
result = get_double(25)

print(f"Using lambda: double of 25 is {result}")

# You could also use it immediately (less common for named operations)
print(f"Using lambda immediately: {(lambda y: y * 2)(30)}")

# Expected Output:
# Using lambda: double of 25 is 50
# Using lambda immediately: 60

Explanation:

  • lambda x: x * 2 defines an anonymous function.

    • lambda: Keyword indicating a lambda function.

    • x: The argument (parameter) the function takes.

    • :: Separates arguments from the expression.

    • x * 2: The single expression whose result will be returned.

  • We assign this lambda function object to the variable get_double.

  • We then call the function using get_double(25), passing 25 as the argument for x. The lambda evaluates 25 * 2 and returns 50.

Solution 10: Function with Loop and Conditional

def find_expensive_items(product_prices_dict, threshold):
    """
    Finds product names with prices above a given threshold.

    Args:
        product_prices_dict (dict): Dictionary of {product_name: price}.
        threshold (float): The price threshold.

    Returns:
        list: A list of names of products exceeding the threshold.
    """
    expensive_products = [] # Initialize empty list to store results

    # Loop through key-value pairs (Module 7 loop, Module 5 dict items)
    for name, price in product_prices_dict.items():
        # Conditional check (Module 6)
        if price > threshold:
            # Append name to the list (Module 4 list method)
            expensive_products.append(name)

    return expensive_products # Return the list of names

# Sample data
products = {"Laptop": 1200.00, "Mouse": 25.00, "Keyboard": 75.00, "Monitor": 250.00}
price_limit = 100.00

# Call the function
high_value_list = find_expensive_items(products, price_limit)

# Print the returned list
print(f"Products costing more than ${price_limit:.2f}: {high_value_list}")

# Expected Output:
# Products costing more than $100.00: ['Laptop', 'Monitor']

Explanation:

  • The function find_expensive_items takes a dictionary and a threshold as input.

  • It initializes an empty list expensive_products to store the results (accumulator pattern for lists).

  • It uses a for loop with .items() to iterate through each product name (key) and price (value) in the input dictionary.

  • Inside the loop, an if price > threshold: statement checks if the current product’s price exceeds the limit.

  • If the condition is true, the product’s name (the key) is added to the expensive_products list using .append().

  • After checking all items in the dictionary, the function returns the expensive_products list, containing the names of all products that met the condition.
